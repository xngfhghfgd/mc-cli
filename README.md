# mc-cli

`mc-cli` 是一个终端里的 Minecraft 无显示客户端，适合做自动化测试、协议实验、基础机器人控制和聊天挂服。

当前仓库已经整理为一个**可直接运行的成品**，重点能力包括：

- 连接服务器并完成登录
- 发送与接收聊天消息
- 支持基础移动控制
- 支持左右键自动连击
- 支持 `ping` 查询服务器状态
- 支持自动识别服务器版本后再连接
- 全程命令行交互，适合脚本化运行

> 说明：当前实现主要面向离线模式服务器；如果服务器要求正版验证/加密登录，会直接提示不支持。

## 目录

- `cli.py`：命令行入口
- `client.py`：连接、登录、聊天、移动、连击逻辑
- `protocol.py`：Minecraft 协议读写与版本适配
- `debug_send.py`：调试辅助脚本
- `mc`：可执行入口

## 快速开始

### 1. 启动客户端

```bash
./mc <服务器地址> [端口] [用户名]
```

例如：

```bash
./mc localhost 25565 HermesBot
```

参数说明：

- `服务器地址`：必填
- `端口`：可选，默认 `25565`
- `用户名`：可选，默认 `HermesBot`

也可以通过环境变量 `MC_USERNAME` 设置默认用户名。

### 2. 只查询服务器状态

```bash
./mc --ping <服务器地址> [端口]
```

示例：

```bash
./mc --ping localhost 25565
```

输出会包含：

- 在线状态
- 延迟
- MOTD
- 在线人数 / 最大人数
- 服务器版本名
- 自动识别结果

### 3. 自动识别版本

```bash
./mc <服务器地址> [端口] [用户名] --auto-version
```

它会先对服务器执行状态探测，然后尽量从返回信息里识别版本，再用识别结果连接。

### 4. 指定版本

```bash
./mc <服务器地址> [端口] [用户名] --version 1.18.2
```

也支持直接填协议号。

## 聊天与控制

启动后可以直接在终端输入文本：

- 普通文本：发送到游戏聊天
- `/quit` 或 `/exit`：退出客户端
- `/aswd on`：开启移动状态
- `/aswd off`：关闭移动状态
- `/click left <间隔> <次数> [延迟] [loop]`：左键连击
- `/click right <间隔> <次数> [延迟] [loop]`：右键连击
- `/stopclick [left|right]`：停止对应连击

### 左右键连击说明

`/click` 命令的格式：

```text
/click left 0.1 5
/click right 0.2 10 0.5 loop
```

字段含义：

- `left|right`：左键或右键
- `间隔`：每次动作之间的时间，最低强制为 `0.05` 秒
- `次数`：每轮动作执行次数
- `延迟`：开始前延迟，默认 `0`
- `loop`：可选，开启后循环执行

例如：

- `/click left 0.1 5`：每 0.1 秒执行 5 次左键动作
- `/click right 0.2 10 0.5 loop`：延迟 0.5 秒后开始，每轮 10 次右键动作，循环执行

## 启动参数连击

如果你希望客户端一连上就开始动作，也可以直接带参数：

```bash
./mc <服务器地址> [端口] [用户名] --click-left 0.1 5
./mc <服务器地址> [端口] [用户名] --click-right 0.2 10
```

## 交互体验

当前终端输入体验包括：

- 输入提示符 `> `
- 本地回显：`[我] 消息`
- 服务器消息回显：`[服] 消息`
- 后台 keepalive / watchdog 任务

## 局限性

- 主要面向离线模式服务器
- 加密登录 / 正版验证服务器暂不支持
- 左右键目前是协议动作层的实现框架，适合自动化测试和实验
- 更完整的实体交互、方块交互、物品使用细分还可以继续扩展
